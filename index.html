HTML

  • The head element contains information about the webpage that helps the browser to render the page correctly.
  • The body element represents the visible content shown to the user
  • Elements that are nested inside other elements are child elements.
  • The body of child elements are known as the parent element
  • Semantic HTML element is where the name of the element describes its contents and the role it plays on a webpage.
  • Elements and tags are used interchangably but element is from opening to closing and tag is inside angle brackets.
  • Doctype HTML: declaration that tells the browser what document type to expect when opened.
  • Declarations are not case sensitive.
  • HTML Elements are case sensitive.
  • Lang= "en": language english
  • HTML attributes are always defined in the beginning element, never the closing element.
  • Title: important for SEO
  • Style: defines CSS styles associated with a website.
  • Base: base URL for a webpage
  • Link: connects external resources to the HTML document.
  • Meta: defines metadata (character set, description, keywords, author, viewport)
  • Href: attribute points to the URL for the link.
  • Elements that are nested inside other elements are child elements. The body is the parent element.
  • Semantic HTML element: the name of the element describes its contents and the role it plays on a webpage.
  • p: paragraph ul: unordered list ol:ordered list li: list items img: image (can contain various attributes) src: location of image br: creates a line of empty space a: anchor (creates links to the same webpage or others)
  • Use headers in order.
  • H1 headings should appear only once on each page.
  • Alt: contains string of text that describes the appearance and function of image.
  • Footer: appears at the bottom of page containing author, copyright, contact, sitemap, and navigation.
  • ID: attribute identifies elements and selects them.
  • Class: attribute that works similarly to ID attribute.

CSS

  • A margin (outside) indicates how much space we want around the outside of an element.
  • A padding (inside) indicates how much space we want around the content inside an element.
  • CSS: Cascading Style Sheets
  • CSS: defines colors, aesthetics, visual layout, and organization.
  • 3 ways to style: Inline, Internal, External Stylesheets
  • Inline: when a dev wants to style an element directly in the HTML file. Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Inline cont.: They are useful for quick and permanent changes but are less flexible than external and internal style sheets because each inline style must be edited individually when you make a design change
  • Internal: embedding an entire stylesheet directly in the HTML file using style element in the head element. The rules only apply to that page and you can configure CSS classes and IDs which are used to identify elements, to style multiple elements in the page code.
  • Internal Note: a single change to the CSS rule will apply to all tagged page elements.
  • External: allows devs to keep all their CSS rules in a separate file, which makes changes easy. WHen using we link the file with the link element placed in the HTML head.
  • Separation of Concerns: based on the idea that every section of code should have its own responsibility. Separation of Concerns refers to separating code into diff files based on use.
  • Incorporate CSS: cd prework-study-guide - cd assets - touch style.css (creates file) - git status - git checkout main (confirms) - git checkout -b feature/add-css (switches branch)
  • Selector: used to apply a rule to all the elements visible on the page.
  • Dry Code: refers to not repeating yourself - so combine 2 or more selectors with same rules. (refer to notes for visual)
  • text-align: one way to center a title or heading on a webpage.
  • display: block - assigns each image to start on a new line.
  • Link element: creates the connection to the external file.
  • Rel attribute: specifies the relationship between the current document and the linked.
  • Href attribute: specifies the location (URL) of the external resource. The href attribute is using a relative path. This means the location is derived from the current location (in this case index.hmtl). To locate the style sheet from the current location, we must look in the assets folder, hence assets/style.css
  • Class attribute: class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in HTML, enabling us to freely choose which element will be assigned the CSS rule.
  • Card = class selector (refer to notes for additional card notes and diagram)

GIT

  • git status: checks what branch we are currently on
  • git checkout -b (branch-name): creates a new branch and switches to it
  • -b: flag creates a new branch
  • git status: to verify
  • git add -A: adds modifications in the current working branch to the staging area.
  • -A: flag indicating we want to add all changes.
  • git commit -m "add brief message": a commit created that captures a snapshot of the currently staged changes.
  • -m: message
  • git pull origin main: doubles checks if local branch is in sync with the base branch in Github.
  • pull: to recieve a branches modifications into the local environment.
  • origin: indicates the source of the pull will be in the Github repo.
  • main: indicates the branch.
  • git push origin feature/starter code: pushes changes we have locally to our remote Github branch.
  • github workflow: create a branch, add commits, open PR, discuss and review, merge and deploy

JAVASCRIPT

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Const: allows us to create variables that can't be reassigned a value. "locks down" When writing JS - we always want to start by declaring variables using const. The second we might use is let.
  • const shape = "circle": shape is the name of our container or variable and the data it stores is circle.
  • Make it a habit to end with semicolon
  • Console.log: method used to output a message to the web console by adding an argument.
  • Style.css: placed with head element.
  • Script.js: placed within body (at bottom)
  • Conditional statements and loops: used to control the order in which code is executed or if its executed at all.
  • If statement: code will only execute if the statement meets a condition or is true.
  • If statement: when computer encounters if statement in JS, it'll check to see if the condition we provide is truthy. If so it'll do whatever action we ask between the curly brackets. If not, computer will ignore whatever is in the curly brackets.
  • Falsy: values of 0, -1, On, "", null, undefined, NaN, or Boolean of false.
  • Truthy: all others including Boolean of true.
  • Else if: second condition
  • === : strict equality operator that checks to see if two values are equal and returns s Boolean result true if values are equal and false if not equal.
  • Array: a group of related data "a single variable used to hold a group of data (typically data that is related im some way)
  • Arrays are useful when we want to perform a task on multiple pieces of data at once.
  • Arrays are not limited to contain only 1 type of data. A single array can store strings, number, and so on.
  • Index: location number (first number in an index is 0 then 1 and so on. Zero Indexed and sequential)
  • Loops: use the predictability pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. In order to use we need shapes index and arrays length. Length tells us how many times to repeat. (.lenghth)
  • Let: allows us to change/reassign the value of the variable over time. Esp important in cases where we need to use the variable to store changing values like when iterating over an array using a for loop.
  • For Loop: starting point- let x=0, condition- as long as the variable x is less than the length of the array, the loop will keep running, final statement- x++ which is a shortcut that allows the array to iterate over each item.
