Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user
  • Elements that are nested inside other elements are child elements.
  • The body of child elements are known as the parent element
  • Semantic HTML element is where the name of the element describes its contents and the role it plays on a webpage.
  • Elements and tags are used interchangably but element is from opening to closing and tag is inside angle brackets.

CSS

  • A margin (outside) indicates how much space we want around the outside of an element.
  • A padding (inside) indicates how much space we want around the content inside an element.
  • CSS: Cascading Style Sheets
  • CSS: defines colors, aesthetics, visual layout, and organization.
  • 3 ways to style: Inline, Internal, External Stylesheets
  • Inline: when a dev wants to style an element directly in the HTML file. Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Inline cont.: They are useful for quick and permanent changes but are less flexible than external and internal style sheets because each inline style must be edited individually when you make a design change
  • Internal: embedding an entire stylesheet directly in the HTML file using style element in the head element. The rules only apply to that page and you can configure CSS classes and IDs which are used to identify elements, to style multiple elements in the page code.
  • Internal Note: a single change to the CSS rule will apply to all tagged page elements.
  • External: allows devs to keep all their CSS rules in a separate file, which makes changes easy. WHen using we link the file with the link element placed in the HTML head.
  • Separation of Concerns: based on the idea that every section of code should have its own responsibility. Separation of Concerns refers to separating code into diff files based on use.
  • Incorporate CSS: cd prework-study-guide - cd assets - touch style.css (creates file) - git status - git checkout main (confirms) - git checkout -b feature/add-css (switches branch)
  • Selector: used to apply a rule to all the elements visible on the page.
  • Dry Code: refers to not repeating yourself - so combine 2 or more selectors with same rules. (refer to notes for visual)
  • text-align: one way to center a title or heading on a webpage.
  • display: block - assigns each image to start on a new line.
  • Link element: creates the connection to the external file.
  • Rel attribute: specifies the relationship between the current document and the linked.
  • Href attribute: specifies the location (URL) of the external resource. The href attribute is using a relative path. This means the location is derived from the current location (in this case index.hmtl). To locate the style sheet from the current location, we must look in the assets folder, hence assets/style.css
  • Class attribute: class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in HTML, enabling us to freely choose which element will be assigned the CSS rule.
